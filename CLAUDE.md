# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What is Hypothex

Hypothex is a runtime debugging MCP server for Claude Code. It runs two concurrent services: a FastAPI HTTP collector (receives logs from instrumented code) and an MCP stdio server (exposes tools to Claude Code). Data is stored in SQLite at `~/.hypothex/hypothex.db`.

## Commands

```bash
# Install (editable, with dev deps)
pip install -e ".[dev]"

# Run all tests
pytest

# Run a single test file
pytest tests/test_db.py

# Run a specific test
pytest -k "test_insert_and_get_logs"

# Run the server (default port 3282)
python -m hypothex.main
```

There is no linter or formatter configured.

## Architecture

Three-layer async architecture, all async/await on top of asyncio:

- **`collector.py`** — FastAPI HTTP server. `POST /log` receives log entries from instrumented user code, validates via Pydantic, writes to DB. Max 1MB payload. Links logs to hypotheses via `hypothesis_ids` field.
- **`mcp_server.py`** — `HypothexMCP(FastMCP)` subclass exposing 8 MCP tools (get_logs, tail_logs, search_logs, list_sessions, clear_session, create_hypothesis, list_hypotheses, update_hypothesis, get_hypothesis_logs). Thin wrappers around DB methods; all return JSON strings.
- **`db.py`** — Async SQLite layer (aiosqlite). Three tables: `logs`, `hypotheses`, `log_hypotheses` (junction). WAL mode, foreign keys enabled. Handles all querying/filtering.
- **`models.py`** — Single `LogEntry` Pydantic model with session_id, level, message, optional data/file/function/line/hypothesis_ids.
- **`main.py`** — Entry point. Starts collector (uvicorn) and MCP server concurrently via `asyncio.gather`. Graceful shutdown via shared `asyncio.Event`. Port configurable via `HYPOTHEX_PORT` env var.

The DB instance is shared: created in `main.py`, passed to both collector and MCP server.

## Testing

- pytest + pytest-asyncio (strict mode)
- All tests use in-memory SQLite (`:memory:`) for isolation
- `test_collector.py` uses `httpx.ASGITransport` + `AsyncClient` against the FastAPI app
- `test_mcp_server.py` calls MCP tools via `server.call_tool()` directly
- No mocking of the database — all tests hit real (in-memory) SQLite

## Key Conventions

- Log levels are constrained to: `debug`, `info`, `warn`, `error`
- Hypothesis IDs follow the pattern `session_id:hN` (auto-generated by DB)
- Hypothesis status values: `pending`, `confirmed`, `rejected`
- Instrumentation in user code uses fire-and-forget HTTP POSTs (try/except to never crash user code)
- `skills/hypothex/SKILL.md` and `skills/hypothex-debug/SKILL.md` define the user-facing debugging workflow — update these when changing MCP tool signatures
